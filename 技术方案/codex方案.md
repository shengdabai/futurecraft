# codex方案（讨论稿｜表格/流程/原型强化版）

> 面向国际与中国青少年用户的 UGC 社区产品全栈技术方案草案  
> 基于当前对话决策整理；后续需对齐 `Futurecraft/TECHNICAL_DOCUMENTATION.md` 的具体业务设定与边界。

## 已确认决策（来自本次讨论）

| 主题 | 决策 |
|---|---|
| 形态 | Web + App；优先做 **iOS 原生应用** |
| 核心内容 | **社区 UGC**（纯帖子流） |
| 社交 | **不允许私聊/群聊**；仅公开发布/评论/留言 |
| 注册/门槛 | **游客模式**；游客可浏览+点赞；发帖/评论/举报/收藏等需登录 |
| 年龄策略 | Global：`13+`；中国区：`14+` |
| 年龄门槛以下 | **仅浏览**（Global <13 / CN <14：不允许注册与互动） |
| 市场节奏 | **海外验证先行**；同时规划中国市场 |
| 登录 | 邮箱 + 第三方：Google / Apple / 微信 |
| 推荐 | **不做算法推荐**；用时间排序/话题/人工精选 |
| 付费 | **按月订阅**（绑定 AI 用量）+ **积分包**（补充 AI 用量） |
| AI | Global：**Gemini**；中国区：**Qwen（通义千问）** |
| 边缘与路由 | **Cloudflare**（CDN/WAF/智能路由层） |
| 外链 | 评论最多 **1 条外链**；**先检后发** + 违规站点过滤（成人/赌博/诈骗等） |
| 账号注销与 UGC | 注销后：历史帖子/评论 **匿名化保留**（审计留痕保留） |

## 产品目标与约束

### 2.1 目标
- 让青少年可以安全地创作、发布、交流与获得正反馈（作品/帖子/话题）。
- 全球可用（i18n、多时区），并可在中国合规上线（数据本地化、内容安全能力）。
- 强风控：未成年人保护、反欺凌、举报/处置/申诉闭环。

### 2.2 关键约束（必须满足）
- iOS 数字内容订阅/积分包：采用 **Apple In-App Purchase (IAP)**（避免审核风险）。
- 不提供私聊：显著降低骚扰/诱导/未成年人风险，但仍需严格社区评论治理。
- 中国与海外：建议 **双环境/双数据域隔离**（同代码不同部署与存储）。
- AI 能力：客户端不直连模型；统一由后端代理（配额/风控/日志/安全），Global 用 Gemini、CN 用 Qwen。

## 核心需求清单（MVP → V1）

### 3.1 MVP（上线最小集）
- 游客浏览：帖子流/帖子详情可浏览；游客仅允许 **点赞**（其余需登录）。
- 账号体系：邮箱注册登录 + Apple/Google 登录；中国区加入微信登录。
- 发帖：文本 + 图片（视频可后置）；话题/标签；草稿；发布前提示与社区规范。
- 评论：楼层评论 + 基础二级回复（可选）；@ 不做或受限；敏感词拦截。
- 互动：点赞/收藏；关注（可选，若引入需风控）。
- 举报：对帖子/评论/用户举报；用户可查看处理状态（简化版）。
- 审核：机审 + 人审后台；处置动作（下架/限流/禁言/封禁）；申诉入口（可选）。
- 订阅与积分：订阅（含试用）+ 积分包；与 **AI 用量**绑定；用量台账与扣减。
- 基础后台：内容列表、举报队列、用户处置、运营配置（话题/置顶/公告）。
- 国际化：至少中英双语框架；内容与 UI 分离。
- AI 问答：订阅/积分扣减、基础安全策略（未成年人提示、拒答、限流）、反馈（👍/👎/举报回答）。

### 3.2 V1（增强）
- 视频/音频内容 + 更完善的媒体处理（转码/封面/水印）。
- 更细颗粒度的未成年人保护：分龄内容可见性、夜间模式限制、陌生人互动限制。
- 运营工具：精选/置顶/活动页、批量审核、风控规则配置、敏感词库版本管理。
- 数据分析：留存/内容供给/举报率/审核时延等指标面板。

## 客户端方案

### 5.1 iOS 应用（优先）
- 技术栈：`Swift` + `SwiftUI`（或 UIKit + SwiftUI 混合），网络 `URLSession`（或 `Alamofire`）。
- 架构：MVVM（或 TCA），支持可测试的状态管理。
- 登录：Apple Sign-In（必做）、Google Sign-In、微信登录（中国区）。
- 媒体：图片选择/压缩上传；视频后续接入转码。
- 付费：StoreKit 2（订阅 + 积分包）。
- 推送（可选）：内容审核结果、评论回复提醒（需合规与用户同意）。

### 5.2 Web（配套）
- 用户 Web（可选）：用于 SEO 展示、活动页、帮助中心、账号管理。
- 后台 Web（建议必做）：运营/审核/风控控制台。
- 技术栈建议：`Next.js` + `TypeScript` + `Tailwind` + `React Query`。

## 后端总体架构

### 6.1 形态选择
- 阶段 1：**模块化单体**（一个后端仓库/服务，按 domain 分模块），降低复杂度。
- 阶段 2：按需拆分（审核、媒体、通知、支付对账、搜索）。

### 6.2 API 形态
- 外部：REST（更易落地、易审计）或 GraphQL（二选一；MVP 更建议 REST）。
- 内部：事件驱动（审核、通知、埋点、媒体处理）。

### 6.3 推荐技术栈（可调整）
- 后端：`NestJS (Node.js/TypeScript)` 或 `FastAPI (Python)`；优先选团队更熟的。
- 主库：`PostgreSQL`
- 缓存/限流：`Redis`
- 对象存储：海外 `S3`，中国 `OSS/COS`（抽象一层 Storage Adapter）
- 搜索：`OpenSearch/Elasticsearch`（中英文检索、违规召回）
- 队列：`RabbitMQ`（简单）或 `Kafka`（规模更大）
- 观测：日志（ELK/OpenSearch）、指标（Prometheus）、链路（OpenTelemetry）

| 模块 | 职责 | 主要存储 |
|---|---|---|
| Auth | 游客会话、登录、绑定第三方、Token | Postgres + Redis |
| Users | 资料、分龄、状态（禁言/封禁） | Postgres |
| Content | 帖子、媒体、话题、可见性/状态 | Postgres + Object Storage |

| AI Gateway | Gemini 代理、配额扣减、日志脱敏、风控 | Postgres + Redis + Queue |
| Admin | 审核/运营控制台 API | Postgres |

## AI 问答系统（Gemini）设计

### 目标（非算法推荐前提下）

| 目标 | 说明 |
|---|---|
| 成本可控 | credits 扣减明确；上限与风控避免被薅 |
| 安全优先 | 未成年人场景：拒绝成人/暴力/自伤引导；提示求助资源 |
| 可运营 | 可配置模型、额度、系统提示词版本、灰度开关 |
| 可审计 | 记录用量与处置，不保存敏感原文或做脱敏存储 |

### 请求链路（表格）

| 步骤 | 后端动作 | 关键控制 |
|---|---|---|
| 1 | 鉴权（必须登录） | 订阅/credits 检查 |
| 2 | 输入过滤 | PII 提示、敏感主题预分类 |
| 3 | 配额扣减预估 | 预留 credits（避免竞态） |
| 4 | Gemini 代理调用 | 超时/重试/熔断 |
| 5 | 输出后过滤 | 违规输出二次拦截与改写/拒答 |
| 6 | 结算扣减 | 按 tokens 实扣、写账本 |
| 7 | 反馈闭环 | 👍/👎/举报回答 → 进入审核队列 |

### 风控与限制（建议默认）

| 项 | 默认建议 |
|---|---|
| 单次最大 tokens | `input+output <= 8,000 tokens`（MVP） |
| 单日 credits 上限 | 订阅用户 `<= 月额度/30*2`，避免一天打爆 |
| 并发 | 每用户并发 1–2 个请求 |
| 频控 | `N requests/min`（按地区与风险评分动态） |
| 违规主题 | 成人内容、赌博、毒品、诈骗、暴力伤害、仇恨、违法教程；自伤给“求助/安全资源”模板 |

### 数据存储策略（建议）

| 数据 | 是否保存 | 建议 |
|---|---:|---|
| 原始问题/回答 | 默认不保存 | 仅保存脱敏摘要/哈希用于排障；如要保存需额外合规与显著告知 |
| 用量记录 | ✅ | `ai_usage`（tokens、credits、模型） |
| 反馈/举报 | ✅ | 作为 UGC 一类进入 `reports/moderation_queue` |

### Provider 抽象（为中国区/成本优化预留）

| 能力 | 建议做法 |
|---|---|
| 接口统一 | 内部定义 `LLMProvider`：`generate(messages, params) -> {text, usage}` |
| 路由策略 | `region + feature_flag + cost_policy` 决定走 Gemini 或 CN Provider |
| 模型配置 | 后台可选 `model_id`，记录在 `ai_usage.model` |
| 可替代供应商（示例） | 中国区：通义千问/智谱/文心等（具体选型后定） |

> 默认：允许中国区“体验等价但约束更强”（模型可替换、限额更严格、能力逐步放开），以确保可上线。

## 核心领域设计（Domain）

### 7.1 账号与身份（Auth）
- 支持游客：设备级 `guest_id`（Keychain/Device storage）+ 服务端短期会话。
- 绑定账号：游客内容可迁移到注册账号（需防滥用：频控 + 风控评分）。
- 身份表：email / apple / google / wechat 统一映射到 `user_id`。

### 7.2 内容（Content）
- `Post`：文本、媒体、话题、标签、可见性（公开/仅订阅/仅审核通过等）。
- `Comment`：层级限制（建议最多 2 层），支持删除/折叠。
- `Reaction`：点赞/收藏。
- `Report`：举报对象（post/comment/user），原因、附言、证据、状态流转。

### 7.3 审核与风控（Moderation）
- 文本：敏感词 + 多语言分类（辱骂/性暗示/暴力/自伤等）。
- 图片/视频：接入第三方内容安全（海外/中国分别选择供应商）。
- 处置：下架、限制曝光、删除、禁言、封禁、强制改昵称/头像。
- 审计：所有处置操作写入 `moderation_actions`，可追溯。

### 7.4 订阅与积分（Billing）
#### 7.4.1 订阅（绑定 AI 用量：Global Gemini / CN Qwen）

| 项 | 建议方案 |
|---|---|
| 订阅价格 | 已确认：`$19.99/月`（可做区域定价） |
| 订阅权益（核心） | 每月包含固定 **AI 额度**（以“AI 积分”或“AI Credits”计） |
| 计费口径 | 后端按模型实际用量（token/字符）换算为 Credits 并扣减（Global Gemini / CN Qwen） |
| 免费体验 | 可配：例如 7 天试用或首月折扣（需你确认） |
| 失败兜底 | 额度不足 → 引导购买积分包或下月刷新；不允许负数透支 |

#### 7.4.1.1 AI Credits 口径（建议先定一个“可运营”的简单模型）

| 项 | 建议 |
|---|---|
| 计量单位 | `1 Credit = 1,000 tokens`（四舍五入到最小 1 Credit 或按 0.1 Credit 计） |
| 扣减方式 | `credits = ceil((input_tokens + output_tokens) / 1000)`（MVP 简单好解释） |
| 订阅月额度 | 已确认：`1,000 Credits/月`（`$19.99/月`） |
| 成本与利润 | 后台按实际供应商账单成本核算，订阅价覆盖成本 + 利润空间 |
| 防滥用 | 单日上限、单次最大 tokens、异常请求熔断与黑名单 |

> 已确认：积分包档位为 `$4.99=250`、`$9.99=600`、`$19.99=1400 Credits`。

#### 7.4.2 积分包（补充 AI 额度）

| 项 | 建议方案 |
|---|---|
| 购买方式 | iOS 使用 IAP Consumable（积分包） |
| 用途 | 仅用于 AI 问答/生成（不用于买曝光，避免社区公平性争议） |
| 台账 | `points_ledger` 记录每次增减、来源（订阅/月度发放/购买/退款回收） |

#### 7.4.3 额外“非 AI”权益（可选，供你挑）

| 权益 | 说明 | 风险 |
|---|---|---|
| 发帖图片上限提升 | 订阅用户单帖可发更多图 | 低 |
| 发帖频次上限提升 | 更高的日发帖/评论上限 | 中（刷屏） |
| 个性化装扮 | 头像框/昵称颜色 | 低 |
| 草稿/收藏上限提升 | 更大的草稿与收藏容量 | 低 |
| 轻运营曝光 | “订阅用户”角标或轻提示（不直接加权排序） | 中（阶层感） |

默认选择（可改）：发帖图片上限提升 + 草稿/收藏上限提升 + 个性化装扮；不做“订阅角标”。

## 数据库设计（初稿）

> 以 PostgreSQL 为主；命名与字段后续可按代码风格调整。

- `users`：id、created_at、status、locale、region、birth_year(optional)、age_bucket、safety_level
- `user_identities`：user_id、provider(email/apple/google/wechat)、provider_user_id、email(optional)
- `sessions`：user_id/guest_id、refresh_token_hash、expires_at、device_info

- `topics`：id、name、locale、status
- `post_topics`：post_id、topic_id
- `comments`：id、post_id、author_id、parent_id(optional)、text、status、created_at
- `reactions`：target_type(post/comment)、target_id、user_id、type(like/bookmark)、created_at
- `reports`：reporter_id、target_type、target_id、reason_code、detail、status、created_at
- `moderation_queue`：target_type、target_id、signals(json)、decision、assignee_id、updated_at
- `moderation_actions`：target_type、target_id、action、operator_id、note、created_at
- `subscriptions`：user_id、platform(ios/web)、status、current_period_end、trial_end
- `iap_receipts`：user_id、product_id、transaction_id、raw_receipt、verified_at
- `points_ledger`：user_id、delta、reason(earned/purchased/spent/refund)、ref_type、ref_id、created_at
- `ai_usage`：user_id、provider(gemini/other)、model、input_tokens、output_tokens、credits_spent、request_id、created_at
- `links`：id、url、normalized_url、domain、risk_level、category、last_checked_at
- `content_links`：target_type(post/comment)、target_id、link_id
- `rate_limits`（可选）：key、window、count（也可放 Redis）

### 关键状态机（表格）

#### 帖子状态 `posts.status`

| 状态 | 含义 | 可见性 |
|---|---|---|
| `draft` | 草稿 | 仅作者 |
| `pending` | 待审核/待外链检测 | 仅作者（建议） |

| 状态 | 含义 |
|---|---|
| `pending` | 待外链检测/待审核 |
| `published` | 可见 |
| `removed` | 已删除/屏蔽 |
